> **用途：** 对比「个人 / 个体户 / 大陆企业 / 香港企业 × 境内 / 境外部署」在 **szbolent.cn 诗词门户 + Looma 商业闭环** 下的可行性与成本，供选型决策。

> **免责声明：** 费用为 **2026 年市场行情区间估算**，非报价单；审核规则以腾讯云、工信部、微信支付、微信公众平台最新政策为准。本文不构成法律或税务意见。

## 1. 决策背景

| 目标 | 说明 |
|------|------|
| 门户 | `szbolent.cn` 静态站（szbolent-portal） |
| API | `api.szbolent.cn` → Looma Flask（诗词、JWT、tier、支付） |
| 小程序 | 微信生态登录 + 浏览 + 付费（PlanetX / Bolent 线） |
| 支付 | P1 目标：微信 JSAPI / 小程序支付；当前 Stub |
| 非目标（本文不展开） | 链上 GenZ 合约、genz.ltd DAO 治理 |

**核心结论（前置）：**

- **大陆用户 + 大陆小程序 + 大陆微信支付** →  practically 需要 **大陆可认经营主体**（有限公司或个体工商户）+ **ICP 备案**（大陆节点）。
- **香港主体 + 境外托管** → 可绕开大陆 ICP，但 **不能直接替代** 大陆微信支付与小程序合法域名闭环。
- **个人主体** → 适合开发与展示，**不适合** 当前商业闭环规划。

## 2. 概念对照（避免混谈）

| 术语 | 管什么 | 认什么证件 |
|------|--------|------------|
| **ICP 备案** | 大陆服务器上对外提供 **`.cn` / 大陆访问** 的网站 | 大陆个人身份证 或 **大陆营业执照** |
| **ICP 经营许可证** | 经营性互联网信息服务（部分类目） | 企业；诗词文化展示类多数 **企业备案即可**，是否需经营许可证视类目与地方口径 |
| **微信小游戏/小程序企业认证** | 小程序发布、支付能力、部分接口 | **大陆营业执照** 或组织机构代码等 |
| **微信支付（大陆商户号）** | 人民币收单、回调、结算 | **大陆营业执照** + 对公/规定结算账户 |
| **WeChat Pay HK** | 港币、香港商户 | **香港 BR**（商业登记证），与大陆商户号 **独立** |
| **公安联网备案** | 备案通过后 30 日内 | 同 ICP 主体 |

**境内 vs 境外（本文定义）：**

| | 境内路径 | 境外路径 |
|---|----------|----------|
| 服务器 | 腾讯云 **大陆节点**（华东/华北等） | 香港、新加坡、美国等节点 |
| 域名示例 | `szbolent.cn`、`api.szbolent.cn` | `genz.ltd`、海外子域 |
| 备案 | **需要** ICP（企业或个人） | **不需要** 大陆 ICP |
| 大陆用户访问 | 正常 | 可能延迟更高；部分服务受限 |
| 微信支付 | 大陆商户号 | 需 HK 商户或第三方，**不能混用** |

## 3. 主体类型 × 能力矩阵

| 能力 | 个人（大陆） | 个体工商户 | 大陆有限公司 | 香港有限公司 |
|------|:------------:|:----------:|:------------:|:------------:|
| ICP 备案 | ✅ 个人备案 | ✅ 企业备案 | ✅ 企业备案 | ❌ 不适用大陆 ICP |
| 经营性网站 / 会员订阅 | ❌ 原则上禁止 | ✅ 有限可行 | ✅ | ⚠️ 仅境外站 |
| 微信小程序（企业能力） | ⚠️ 受限 | ✅ | ✅ | ⚠️ 需大陆关联或境外小程序策略 |
| 微信支付（大陆） | ❌ | ✅ | ✅ | ❌ 需另设大陆主体 |
| 微信商户发票 / 合同 | ❌ | ⚠️ 部分 | ✅ | 香港票据体系 |
| 对公账户 | ❌ | ✅ 个体户账户 | ✅ | ✅ 香港账户 |
| 品牌对外签约 | 弱 | 中 | 强 | 强（境外） |
| 后续融资 / 股权 | ❌ | 弱 | ✅ | ✅ |
| 维护复杂度 | 低 | 中 | 中高 | 中（秘书、年报） |

## 4. 五条实现路径（详述）

### 路径 A — 大陆有限公司 + 大陆部署（推荐 · 与现有 P0–P2 一致）

**架构：**

```text
szbolent.cn / api.szbolent.cn（备案）
  → 腾讯云大陆 CVM/Lighthouse
  → Looma :5200 + Nginx
  → 微信小程序 + 微信支付（同主体）
```

| 项 | 内容 |
|----|------|
| 适用 | 正式商业运营、会员订阅、小程序、对公收款 |
| 备案 | 企业 ICP；`szbolent.cn` + `api.szbolent.cn` 一并提交 |
| 支付 | 申请大陆微信支付商户号；`PAYMENT_STUB_MODE=false` |
| 周期 | 执照 1–2 周（若新设）+ 备案 7–20 工作日 + 微信商户 3–10 工作日 |
| 风险 | 类目选错导致小程序/商户审核补材料 |

---

### 路径 B — 大陆个体工商户 + 大陆部署（轻量商业）

**与路径 A 的差异：**

| 项 | 有限公司 | 个体工商户 |
|----|----------|--------------|
| 注册成本 | 较高 | **较低** |
| 税务 / 账务 | 规范要求高 | 相对简单（核定或查账因地而异） |
| 微信支付 | ✅ | ✅（多数文化/服务类目可申请） |
| 品牌与融资 | 更适合长期 | 适合个人品牌试水 |
| 发票 | 可开 | 可开（能力因地区而异） |
| 小程序企业认证 | ✅ | ✅ |

**适用：** Jason 个人品牌诗词门户 **先验证付费**，再升级有限公司。

---

### 路径 C — 个人主体 + 大陆备案（仅展示 / 开发）

**架构：** 个人 ICP + 大陆服务器；门户只展示诗词，**不接真实支付**。

| 项 | 内容 |
|----|------|
| 适用 | 备案域名先行、内容审核、本地/内网联试 |
| 支付 | ❌ 无大陆商户号；Looma 保持 `PAYMENT_STUB_MODE` 或内测 |
| 小程序 | 个人小程序能力受限；**不宜** 作为商业主载体 |
| 升级 | 后续变更主体需 **备案变更** + 小程序/商户重新绑定 |

**适用：** 「不急着商业闭环、先占坑域名与内容」——与你们「等备案后再联试」兼容，但 **不能作为终态**。

---

### 路径 D — 香港公司 + 境外托管（genz.ltd 线）

**架构：**

```text
genz.ltd（无大陆 ICP）
  → 腾讯云香港 / Cloudflare / 其他境外节点
  → 可选 Stripe / PayPal / WeChat Pay HK
```

| 项 | 内容 |
|----|------|
| 适用 | 海外用户、英文产品、GenZ 品牌、链上叙事 |
| 大陆 ICP | **不需要** |
| szbolent.cn | **不能** 在同一架构下替代（.cn + 大陆用户预期 → 仍要路径 A/B） |
| 微信支付 | WeChat Pay HK（港币）；**大陆微信用户无法走同一套人民币小程序支付** |
| 小程序 | 若服务大陆用户，仍面临合法域名与主体问题 |

**适用：** 与 Bolent **叙事分离** 的境外产品；**不是** `TENCENT_CLOUD_COMMERCE.md` P0 的 drop-in 替代。

---

### 路径 E — 混合：香港母公司 + 大陆子公司（**团队既定结构**）

> **现状：** 已有香港公司 + 已有大陆公司（香港公司持股大陆公司）；**无**大陆个体户。  
> **结论：** 对 szbolent.cn + 微信生态 **合理且推荐**；Bolent 大陆闭环走 **大陆子公司**，genz.ltd 走 **香港母公司**。

**股权与职能架构：**

```text
┌─────────────────────────────────────────────────────────────┐
│  香港有限公司（母公司）                                        │
│  · 持股大陆子公司                                             │
│  · 品牌 / IP / 境外合同（genz.ltd、WeChat Pay HK 等）          │
│  · 秘书、年报、BR 续期                                        │
│  · 不参与大陆 ICP 备案与大陆微信支付                           │
└──────────────────────────┬──────────────────────────────────┘
                           │ 持股
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  大陆有限公司（子公司 · 境内运营主体）                          │
│  · ICP 备案：szbolent.cn、api.szbolent.cn                     │
│  · 腾讯云大陆服务器、Looma API、门户静态站                      │
│  · 微信小程序企业认证、微信支付商户号、对公收款                    │
│  · 隐私政策 / 用户协议「运营者」、发票、劳动用工（如有）          │
└──────────────────────────┬──────────────────────────────────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                 ▼
   szbolent.cn      api.szbolent.cn      微信小程序
   (portal dist)    (Looma :5200)       (wx.login / pay)
```

**双域分工（避免混线）：**

| 域 / 产品 | 法律主体 | 部署 | 支付 |
|-----------|----------|------|------|
| `szbolent.cn`、`api.szbolent.cn` | **大陆子公司** | 腾讯云 **大陆** 节点 | 大陆微信支付 |
| `genz.ltd` 及境外子域 | **香港母公司** | 香港 / 海外节点 | WeChat Pay HK 或 Stripe 等 |
| 本地开发 `localhost` | — | 本机 | Stub / 不测支付 |

**与路径 A 的关系：** 大陆子公司在能力上等同 **路径 A**；路径 E 额外多一层香港控股与跨境合规，**无需再注册个体户**。

| 项 | 内容 |
|----|------|
| 适用 | 已有 HK + 大陆双主体；境内小程序 + 人民币 + 境外品牌/IP |
| 备案 | **大陆子公司** 企业 ICP |
| 支付 | **大陆子公司** 微信商户号；香港公司 **不** 替代 |
| 周期 | 主体已存在 → 主要等备案 7–20 工作日 + 商户 3–10 工作日 |
| 复杂度 | 中高（HK 年报 + 大陆账务；IP/许可费跨境需税务规划） |
| 相对路径 B | 已有大陆法人，**优于** 新设个体户 |

**适用：** 当前团队结构；执行 `TENCENT_CLOUD_COMMERCE.md` P0–P2 时以 **大陆子公司** 为唯一对外运营主体。

## 5. 既定架构合规要点（香港持股 · 大陆运营）

> 本节对应 **路径 E**，供备案、小程序、商户申请与上线前自检。不构成法律或税务意见。

### 5.1 主体「三一致」原则（大陆侧 · 必守）

以下场景的 **主体名称、统一社会信用代码** 必须指向 **同一大陆子公司**：

| # | 场景 | 常见错误 |
|---|------|----------|
| 1 | ICP 备案（腾讯云备案系统） | 误填香港公司名 |
| 2 | 微信小程序企业认证 | 与备案主体不一致 |
| 3 | 微信支付商户入驻 | 与小程序主体不一致 |
| 4 | 门户 Footer / 隐私政策 / 用户协议「运营者」 | 写品牌名但未列法人 |
| 5 | 腾讯云大陆服务器购买合同 | 个人账号 vs 企业主体混用 |
| 6 | 公安联网备案（ICP 通过后 30 日内） | 遗漏或主体不符 |

**检查口令：** 备案证、小程序后台、商户平台、网站页脚 **四处名称一致**。

### 5.2 香港母公司 · 职责边界

| 可做 | 不可做（须由大陆子公司） |
|------|---------------------------|
| 持有大陆子公司股权 | 大陆 ICP 备案 |
| 持有商标、IP、genz.ltd 品牌 | 大陆微信支付商户号 |
| 境外合同、WeChat Pay HK | 小程序企业认证（大陆版） |
| 向大陆子公司许可品牌 / 技术（需合同 + 税务处理） | 直接作为 `szbolent.cn` 运营者公示 |

### 5.3 大陆子公司 · 上线前自检清单

**证照与账户**

- [ ] 营业执照在有效期内
- [ ] 确认公司类型：**内资有限公司** 或 **WFOE（外商独资）**（影响经营范围表述，见 §5.4）
- [ ] 大陆对公银行账户可用（微信支付结算）
- [ ] 法定代表人、网站负责人身份证就绪（备案用）

**经营范围**

- [ ] 含 **互联网信息服务** 或地方市监认可的等效表述（诗词文化展示 / 信息服务等）
- [ ] 若缺项：评估 **增项变更** 后再提交 ICP / 商户（周期约 1–3 周，因地而异）

**备案材料（`szbolent.cn` + `api.szbolent.cn`）**

- [ ] 网站名称与小程序名称协调（如 SZBolent 诗词门户）
- [ ] 首页 URL：`https://szbolent.cn`
- [ ] 隐私政策、用户协议可访问（可先静态页 `/legal/*`）
- [ ] 服务器在大陆节点且 IP 与备案一致

**微信生态**

- [ ] 小程序 AppID 企业认证主体 = 大陆子公司
- [ ] 商户平台绑定 **同一** 小程序 AppID
- [ ] 公众平台配置 `api.szbolent.cn` 为 request 合法域名（P0-5）
- [ ] 支付回调 URL：`https://api.szbolent.cn/v1/payment/notify/wechat`（P1）

**代码与配置对齐**

- [ ] `szbolent-portal` 生产构建：`VITE_LOOMA_API_BASE=https://api.szbolent.cn`
- [ ] `src/config/company.ts`：运营者、备案号、联系方式与 **大陆子公司** 一致
- [ ] Looma 生产 `.env`：`WECHAT_PAY_NOTIFY_URL` 等指向备案域名

### 5.4 WFOE 与内资子公司（若适用）

| 类型 | 备案 / 支付 | 额外注意 |
|------|-------------|----------|
| **内资有限公司** | 与一般路径 A 相同 | 经营范围按当地口径 |
| **WFOE（外商独资）** | 可用大陆执照做 ICP 与微信商户 | 股东为境外（香港）公司；部分类目有外资准入限制；诗词文化展示类 **通常可行**，以当地市监为准 |
| **个体工商户** | 团队 **未采用** | 已有大陆法人，**无需新设** |

### 5.5 跨境合规（HK ↔ 大陆 · 中长期）

| 项 | 说明 | 建议 |
|----|------|------|
| 品牌 / IP 许可 | 香港公司 → 大陆子公司 | 书面许可协议；定价符合独立交易原则 |
| 管理费 / 技术服务费 | 跨境收付 | 税务与外汇合规；咨询代理记账 / 税务顾问 |
| 利润汇出 | 大陆 → 香港 | 按企业所得税、分红及外汇规定办理 |
| 两套年报 | HK 秘书年报 + 大陆工商年报 | 日历提醒，避免逾期 |

### 5.6 与项目节奏的对齐

| 阶段 | 香港母公司 | 大陆子公司 |
|------|------------|------------|
| **现在（备案审核中）** | 维持秘书 / BR；genz 线不阻塞 Bolent | 提交 / 等待 ICP；并行准备商户材料 |
| **备案通过后** | 无额外动作（Bolent 线） | 执行 P0：DNS、SSL、小程序合法域名 |
| **P1 支付** | 无 | 商户号、回调、0.01 元实单 |
| **genz.ltd 单独立项** | 境外站 + HK 支付 | 不混用 szbolent 域名与商户号 |

### 5.7 架构合理性结论

| 问题 | 结论 |
|------|------|
| 香港持股 + 大陆运营是否合理？ | **是**，为跨境常见结构 |
| 是否需要大陆个体户？ | **否**，大陆子公司已覆盖 |
| Bolent P0–P2 执行主体？ | **大陆子公司** |
| genz.ltd 执行主体？ | **香港母公司**（与 Bolent 分域） |
